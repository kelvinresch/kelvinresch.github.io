Time of Flight Sensors

Prelab

Sensor I2C
ddresses

This is a Time-of-Flight sensor. It works by emitting a light pulse and then measuring the time it takes for that pulse to come back. In order to use this sensor, I need to hook it up to my Artemis microcontroller over I2C. According to the Pololu site for the VL53L1X Time-of-Flight Sensor, the sensor's 7-bit I2C address is 1010010b, which makes sense because there is a private int called "_i2cAddress = 0x52" in Sparkfun's library header file, which corresponds to that address.

When I print the I2C address into serial monitor from the sensor, I get 82, which is 0x52 in hex and 1010010b in binary.

2 ToF Sensors

Since every sensor defaults to that address, and I want more than one sensor, I need to be able to write addresses to my sensors. I can do this using Sparkfun's "setI2CAddress()" method. This way, I can create two SFEVL53L1X objects in my code, each with different addresses so that the sensors know who I'm requesting data from, even if they are on the same data line.
